## 1. 项目概述

### 1.1 背景与目的

用户（老板）自己写完文章后，不知道文章是否存在问题或可以改进的地方。希望通过辩论机制，让多个专业角色对文章进行分析，发现优点和问题。

### 1.2 核心价值

- **多视角审视**: 不同角色的批评者从各自专长角度分析
- **辩论机制**: 批评者 vs 辩护者的对抗性讨论，更容易暴露问题
- **结构化输出**: 标准化的评分和建议，便于作者改进
- **报告导出**: 支持 PDF/HTML 格式导出
- **灵活配置**: 支持自定义角色、数量和辩论轮次

## 4. 角色设计

### 4.1 批评者 (Critic)

**职责**: 严格审视文章，发现问题

**已实现的批评者**:

| 角色 | 专注领域 | 状态 |
|------|----------|------|
| **结构批评者** | 段落安排、逻辑框架、过渡衔接 | ✅ |
| **逻辑批评者** | 论证严密性、因果关系、证据充分性 | ✅ |
| **语言批评者** | 用词准确、句式变化、表达流畅 | ✅ |
| **创意批评者** | 立意深度、独特视角、思想价值 | ✅ |

| **商业批评者** | 市场定位、竞争策略、商业价值 | 🔄 |

### 4.2 辩护者 (Defender)

**职责**: 为文章优点辩护，反驳不合理的批评

|------|----------|------|
| **平衡辩护者** | 理性分析，寻求共识 | ✅ |
| **共情辩护者** | 理解作者意图，为创作语境辩护 | ✅ |
| **内容辩护者** | 为内容和观点辩护 | 🔄 |
| **表达辩护者** | 为表达方式和风格辩护 | 🔄 |

### 4.3 编辑 (Editor)

**职责**: 综合评分，给出修改建议 | ✅ 已实现

## 5. 辩论流程

### 5.1 可配置参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--rounds` | 辩论轮次 | 3 |
| `--critics` | 批评者列表/数量 | 2 |
| `--defenders` | 辩护者列表/数量 | 2 |

### 5.2 标准流程

```
Round 1: 批评者初审 → 问题清单
         辩护者回应 → 优点辩护

Round 2: 批评者复审 → 坚持/修正观点
         辩护者再回应 → 深化辩护

Round 3: 批评者终审 → 最终问题清单
         辩护者总结 → 核心立场

Round N: 编辑综合 → 最终报告
```

## 7. 数据结构

### 7.1 项目 (Project)

```typescript
interface Project {
  id: string;
  title: string;
  article: string;
  createdAt: string;
  config: {
    rounds: number;
    critics: string[];  // 选中的批评者角色列表
    defenders: string[]; // 选中的辩护者角色列表
  };
  status: 'pending' | 'debating' | 'completed';
  rounds: RoundResult[];
  finalReport: FinalReport;
}
```

### 7.2 轮次结果 (RoundResult)

```typescript
interface RoundResult {
  roundNum: number;
  critics: AgentView[];
  defenders: AgentView[];
}
```

### 7.3 Agent观点 (AgentView)

```typescript
interface AgentView {
  name: string;      // 如"批评者-A"、"批评者-B"
  role: string;       // 如"结构批评者"
  content: string;     // 评审/辩护内容
}
```

### 7.4 最终报告 (FinalReport)

```typescript
interface FinalReport {
  score: number;           // 综合评分
  dimensions: {
    name: string;           // 维度名称
    score: number;          // 评分
    comment: string;         // 简评
  }[];
  pros: string[];            // 优点列表
  cons: string[];             // 问题列表
  suggestions: {
    must: string[];          // 必须修改
    should: string[];       // 建议修改
    optional: string[];      // 可选修改
  };
  verdicts: {                // 裁决结果
    issue: string;           // 争议点
    critic: string;          // 批评者观点
    defender: string;        // 辩护者观点
    ruling: string;          // 编辑裁决
  }[];
}
```

```bash

# 基本用法
python3 debate.py "文章内容"
```

## 10. 用户交互流程

### 10.1 创建项目流程

```
1. 点击"上传文章"按钮
   ↓
2. 填写文章标题（自动从文件名提取或手动输入）
   ↓
3. 选择输入方式：
   - 粘贴文本：直接输入文章内容
   - 上传文件：支持 TXT, MD, PDF, DOC, DOCX
   ↓
4. 配置辩论参数：
   - 辩论轮次：1-5轮
   - 批评者角色：多选（至少1个）
   - 辩护者角色：多选（至少1个）
   ↓
5. 点击"开始辩论"创建项目
```

### 10.2 查看项目流程

```
1. 在项目列表页点击项目卡片
   ↓
2. 进入项目详情页
   ↓
3. 查看辩论内容：
   - 按轮次浏览批评者/辩护者观点
   - 使用筛选器查看特定角色
   - 展开/折叠Agent卡片查看详细内容
   ↓
4. 查看评审报告：
   - 综合评分和雷达图
   - 优点/问题/建议
   - 裁决结果
   ↓
5. 导出报告：点击"导出报告"下载PDF/HTML
```

### 10.3 删除项目流程

```
1. 悬停在项目卡片上
   ↓
2. 点击右上角删除按钮（🔴垃圾桶图标）
   ↓
3. 确认删除对话框弹出
   ↓
4. 点击"确认"删除项目
```
